Any ENS name can issue subdomains to itself, other people, or smart contracts. If you own `alice.eth`, you can create `pay.alice.eth`, `dao.alice.eth`, or `bob.alice.eth`. Subnames are full ENS names — they can have their own resolvers, records, and even their own subdomains.

This composability makes subnames a powerful primitive: issue named credentials, organize wallets, build a community namespace, or let your protocol's contracts be discoverable by name.

---

## Types of Subdomains

ENS subnames come in three flavors, each with different trust, cost, and complexity tradeoffs.

### L1 Onchain Subnames

Created directly on Ethereum Mainnet through the ENS Registry or Name Wrapper. These are the most straightforward — the full state lives onchain — but gas fees apply for every issuance.

Use L1 subnames when:
- You're creating a small number of names for yourself or your team.
- You want the strongest trustlessness guarantees.
- Gas cost per issuance is acceptable.

### L2 Subnames

Your parent name on L1 delegates resolution to a smart contract on an L2. Subname data lives on the L2, dramatically reducing issuance costs while keeping the system trustless through [CCIP Read](https://docs.ens.domains/resolvers/ccip-read).

[Durin](https://durin.dev/) provides an opinionated, ready-to-use stack for issuing ENS subnames on L2 networks.

### Offchain Subnames

Subname records are stored in a centralized database and served via a CCIP-Read gateway. From a resolution perspective, users can't tell the difference from onchain names. These offer the lowest cost and highest throughput, making them ideal for consumer apps that need to name many addresses quickly.

Providers like [NameStone](https://namestone.com/), [Namespace](https://namespace.ninja/), and [JustaName](https://justaname.id/) offer APIs for offchain subname management.

## Creating Subnames

### For Unwrapped Names (ENS Registry)

If your name is **not** wrapped in the Name Wrapper, create subnames by calling `setSubnodeRecord` on the ENS Registry:

```ts
import { createWalletClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { labelhash, namehash, normalize } from 'viem/ens'

const walletClient = createWalletClient({ chain: mainnet, transport: http() })

const ENS_REGISTRY = '0x00000000000C2E074eC69A0dFb2997BA6C7d2e1e'

await walletClient.writeContract({
  address: ENS_REGISTRY,
  abi: [
    {
      name: 'setSubnodeRecord',
      type: 'function',
      inputs: [
        { name: 'node', type: 'bytes32' },    // parent namehash
        { name: 'label', type: 'bytes32' },   // labelhash of new subdomain label
        { name: 'owner', type: 'address' },   // owner of the new subdomain
        { name: 'resolver', type: 'address' }, // resolver for the new subdomain
        { name: 'ttl', type: 'uint64' },      // TTL
      ],
    },
  ],
  functionName: 'setSubnodeRecord',
  args: [
    namehash(normalize('alice.eth')),         // parent
    labelhash('pay'),                          // label for 'pay.alice.eth'
    '0xSubdomainOwnerAddress',
    '0x231b0Ee14048e9dCcD1d247744d114a4EB5E8E63', // Public Resolver
    0n,
  ],
})
```

### For Wrapped Names (Name Wrapper)

If your name is wrapped in the Name Wrapper, use its `setSubnodeRecord` function instead:

```ts
import { createWalletClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { labelhash, namehash, normalize } from 'viem/ens'

const NAME_WRAPPER = '0xD4416b13d2b3a9aBae7AcD5D6C2BbDBE25686401'

await walletClient.writeContract({
  address: NAME_WRAPPER,
  abi: [
    {
      name: 'setSubnodeRecord',
      type: 'function',
      inputs: [
        { name: 'parentNode', type: 'bytes32' },
        { name: 'label', type: 'string' },      // plain label string (not hash)
        { name: 'owner', type: 'address' },
        { name: 'resolver', type: 'address' },
        { name: 'ttl', type: 'uint64' },
        { name: 'fuses', type: 'uint32' },      // Name Wrapper fuse bits
        { name: 'expiry', type: 'uint64' },     // expiry timestamp
      ],
    },
  ],
  functionName: 'setSubnodeRecord',
  args: [
    namehash(normalize('alice.eth')),
    'pay',                                      // label as plain string
    '0xSubdomainOwnerAddress',
    '0x231b0Ee14048e9dCcD1d247744d114a4EB5E8E63',
    0n,
    0,                                          // no fuses burned
    0n,                                         // no expiry (inherits parent)
  ],
})
```

<Note>
  To check whether a name is wrapped, call `ownerOf(namehash(name))` on the Name Wrapper contract. If it returns a non-zero address, the name is wrapped.
</Note>

## Building a Subname Registrar

If you want to let users claim subnames under your name (e.g., `*.yourprotocol.eth`), you can deploy a subname registrar contract. The pattern is:

1. Deploy a registrar contract.
2. Transfer manager (controller) rights over your parent name to the registrar contract.
3. Users call the registrar's registration function, which calls `setSubnodeRecord` on their behalf.

Here's a minimal registrar example:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

interface IENSRegistry {
    function setSubnodeRecord(
        bytes32 node,
        bytes32 label,
        address owner,
        address resolver,
        uint64 ttl
    ) external;
}

interface IPublicResolver {
    function setAddr(bytes32 node, address addr) external;
    function multicall(bytes[] calldata data) external returns (bytes[] memory);
}

contract SubnameRegistrar {
    IENSRegistry constant registry =
        IENSRegistry(0x00000000000C2E074eC69A0dFb2997BA6C7d2e1e);

    address constant resolver =
        0x231b0Ee14048e9dCcD1d247744d114a4EB5E8E63;

    bytes32 public immutable parentNode;  // namehash of parent name
    address public owner;

    mapping(bytes32 => bool) public taken;

    constructor(bytes32 _parentNode) {
        parentNode = _parentNode;
        owner = msg.sender;
    }

    /// @notice Register a subname for the caller
    /// @param label The subdomain label (e.g., "bob" for bob.parent.eth)
    function register(string calldata label) external {
        bytes32 labelHash = keccak256(bytes(label));
        require(!taken[labelHash], "Label already taken");

        taken[labelHash] = true;

        // Create the subname and assign it to the caller
        registry.setSubnodeRecord(
            parentNode,
            labelHash,
            msg.sender,    // subname owner
            resolver,
            0              // TTL
        );
    }

    /// @notice Allow owner to reclaim control of a subname
    function reclaim(string calldata label, address newOwner) external {
        require(msg.sender == owner, "Not authorized");
        bytes32 labelHash = keccak256(bytes(label));
        registry.setSubnodeRecord(parentNode, labelHash, newOwner, resolver, 0);
    }
}
```

<Steps>
  <Step title="Deploy the registrar contract">
    Deploy the contract and note its address. Pass the `namehash` of your parent name to the constructor.
  </Step>
  <Step title="Grant manager rights to the registrar">
    In the ENS app, go to your name → **More** → **Send Manager** and enter the registrar contract's address. This allows the contract to create subnames under your name.
  </Step>
  <Step title="Users call register()">
    Users can now call `register("bob")` on your registrar contract and receive `bob.yourname.eth`.
  </Step>
</Steps>

<Warning>
  Once you transfer manager rights to a contract, you lose direct control over subname creation. Make sure your contract includes owner controls to reclaim names or update the setup if needed.
</Warning>

## Use Cases

| Use Case | Example |
|---|---|
| **Team namespacing** | `dev.myprotocol.eth`, `ops.myprotocol.eth` |
| **User identity** | `alice.app.eth`, `bob.app.eth` |
| **Contract naming** | `vault.protocol.eth`, `treasury.dao.eth` |
| **Wallet organization** | `savings.alice.eth`, `trading.alice.eth` |
| **NFT projects** | Each token holder gets `tokenid.project.eth` |
| **DAO membership** | `member.dao.eth` issued on governance participation |

## Checking if a Subdomain Exists

Before creating a subdomain, check whether the label is already registered:

```ts
import { createPublicClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { labelhash, namehash, normalize } from 'viem/ens'

const client = createPublicClient({ chain: mainnet, transport: http() })

const ENS_REGISTRY = '0x00000000000C2E074eC69A0dFb2997BA6C7d2e1e'

const subnameNode = namehash(normalize('pay.alice.eth'))

const currentOwner = await client.readContract({
  address: ENS_REGISTRY,
  abi: [
    {
      name: 'owner',
      type: 'function',
      inputs: [{ name: 'node', type: 'bytes32' }],
      outputs: [{ name: '', type: 'address' }],
    },
  ],
  functionName: 'owner',
  args: [subnameNode],
})

const isAvailable = currentOwner === '0x0000000000000000000000000000000000000000'
console.log('Available:', isAvailable)
```

<Tip>
  For large-scale subname issuance, consider L2 or offchain subnames to avoid mainnet gas costs. Tools like [Durin](https://durin.dev/) make L2 subname issuance straightforward without needing to build the CCIP Read infrastructure yourself.
</Tip>
