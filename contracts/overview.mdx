The Ethereum Name Service is built on a set of purpose-built smart contracts deployed on Ethereum Mainnet. Each contract has a distinct responsibility: the **Registry** stores ownership records, **Registrars** govern how names are claimed, **Resolvers** answer lookup queries, and the **Name Wrapper** adds fine-grained permission control via ERC-1155 NFTs.

## Architecture at a Glance

<CardGroup cols={2}>
  <Card title="Registry" icon="database" href="/contracts/registry">
    The root contract that maps every ENS name (as a namehash node) to an owner, resolver, and TTL. All ENS lookups start here.
  </Card>
  <Card title="ETH Registrar" icon="ethereum" href="/contracts/eth-registrar">
    Manages registration and renewal of `.eth` second-level domains. Implements a commit-reveal scheme and Dutch-auction premium pricing.
  </Card>
  <Card title="DNS Registrar" icon="globe" href="/contracts/dns-registrar">
    Lets you claim DNS names (e.g. `example.com`) in ENS by submitting DNSSEC proofs, with an optional gasless path via CCIP-Read.
  </Card>
  <Card title="Reverse Registrar" icon="arrow-left-right" href="/contracts/reverse-registrar">
    Maps Ethereum addresses back to ENS names, powering the "primary name" feature on Mainnet and popular L2s.
  </Card>
  <Card title="Resolvers" icon="magnifying-glass" href="/contracts/resolvers">
    Store and serve records — addresses, text, content hashes, and more. Includes the Public Resolver, Universal Resolver, and CCIP-Read resolvers.
  </Card>
  <Card title="Name Wrapper" icon="box" href="/contracts/namewrapper">
    Wraps any ENS name as an ERC-1155 NFT and enforces permissions through a fuse-based system.
  </Card>
</CardGroup>

## How Resolution Works

When a client resolves a name like `alice.eth`, the flow is:

<Steps>
  <Step title="Query the Registry">
    Call `registry.resolver(namehash('alice.eth'))` to find which resolver contract is assigned to the name.
  </Step>
  <Step title="Query the Resolver">
    Call the resolver's method for the record you need — for example `addr(node)` for an ETH address or `text(node, 'twitter')` for a text record.
  </Step>
  <Step title="Handle CCIP-Read (if needed)">
    If the resolver reverts with an `OffchainLookup` error, follow the EIP-3668 callback protocol to fetch data from an L2 or offchain gateway.
  </Step>
</Steps>

The **Universal Resolver** combines all three steps into a single contract call, making it the recommended entry point for library developers.

## TLD Ownership

Each top-level domain is governed by a dedicated registrar contract:

| TLD | Registrar |
|---|---|
| `.eth` | ETH Registrar |
| `.com`, `.xyz`, and other DNS names | DNS Registrar |
| `.addr.reverse` | Reverse Registrar |
| `[root]` | ENS Registry (owned by ENS DAO) |
